launch_acp_docker¶

ansys.acp.core.launch_acp_docker(*, image_name='ghcr.io/pyansys/pyacp-private:latest', license_server, mount_directories=mappingproxy({}), port=None, stdout_file='/dev/null', stderr_file='/dev/null')¶

Launch an ACP server locally in a Docker container.

Use docker run to locally start an ACP Docker container.

Parameters

  • image_name (str) – The name of the Docker image to launch.

  • license_server (str) – The ANSYSLMD_LICENSE_FILE environment variable that is passed to the Docker container.

  • mount_directories (Mapping[str, str]) – Local directories which should be mounted to the Docker container. The keys contain the path in the context of the host, and the values are the paths as they should appear inside the container.

  • stdout_file (Union[str, os.PathLike]) – Path (on the host) to which the output of docker run is redirected.

  • stderr_file (Union[str, os.PathLike]) – Path (on the host) where the standard error of docker run is redirected.

  • port (Optional[int]) –

Returns

Server object which can be used to control the server, and instantiate objects on the server.

Return type

ansys.acp.core._server.ServerProtocol
